# Documentation

This directory contains project documentation templates for quickly bootstrapping new work.

## Structure
- **DESIGN_OVERVIEW.md** - Captures goals, scope, risks, and MVP definition before building.
- **ARCHITECTURE.md** - High-level system design, component model, and operational concerns.
- **API.md** - External contracts and schemas with testing hooks.
- **SETUP.md** - Local development prerequisites and step-by-step environment setup.
- **DEPLOYMENT.md** - Deployment and release process, including rollback guidance.
- **TROUBLESHOOTING.md** - Common issues and fast diagnostics.
- **EXAMPLES.md** - Runnable examples and walkthroughs of key flows.
- **ONBOARDING_CHECKLIST.md** - Sequential steps to bootstrap a new fork using the agentic workflow.
- **ISSUE_WORKFLOW.md** - How to use GitHub issues/comments to pass context between agents.
- **reference/** - Curated code snippets, API references, and third-party data notes for quick reuse.

## For AI Agents
When generating documentation:
1. Keep it practical and actionable.
2. Use copy/pasteable commands and examples where possible.
3. Update docs whenever functionality changes.
4. Write for both human and AI readers.
5. Use clear, unambiguous language and prefer checklists.
